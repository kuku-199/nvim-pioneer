## 1. 这是什么

这是一套个人定制的 Neovim 配置，兼顾**功能完整**和**手感流畅**。

**一句话总结**：打开后按 `<Space>`（空格）是大部分操作的起点。

**核心功能一览：**

| 功能 | 用什么实现的 |
|------|------------|
| 文件搜索 | telescope.nvim（模糊搜索，带预览 + fzf 排序） |
| 代码补全 | blink.cmp（Rust 编写，极速） |
| 代码智能 | nvim-lspconfig + Mason（自动装 LSP） |
| 语法高亮 | nvim-treesitter（基于语法树，更准） |
| 闪电跳转 | flash.nvim（按 `s` 跳到任意可见字符） |
| 诊断中心 | trouble.nvim（`<leader>xx` 一键看所有问题） |
| 全局替换 | nvim-spectre（`<leader>sr` 面板式替换） |
| 格式化 | conform.nvim（保存自动格式化） |
| 文件浏览 | neo-tree + oil（双模式文件管理） |
| 顶部标签栏 | bufferline.nvim（诊断标记 + 文件图标） |
| 命令行美化 | noice.nvim（圆角浮动 cmdline + mini 消息） |
| 窗口标题栏 | incline.nvim（每窗口顶部浮动文件名） |
| 动画系统 | mini.animate（光标轨迹 + 平滑滚动） |
| Git 集成 | gitsigns（行内改动标记） |
| 会话管理 | persisted.nvim（退出自动保存，启动自动恢复） |
| 主题切换 | 4 套主题循环（gruvbox / nightfox / rose-pine / dracula） |
| 调试 | nvim-dap（F5/F10/F11/F12 调试） |
| 状态栏 | lualine（CPU/RAM/venv/Copilot/LSP 状态） |
| Markdown | 浏览器预览 + 编辑器内渲染 |

## 2. 基本概念：Mode 模式

Neovim 跟 VS Code/Eclipse 最大的不同是它有**模式**：

| 模式 | 说明 | 什么时候用 |
|------|------|-----------|
| **普通模式** Normal | 按 `Esc` 进入。移动光标、复制粘贴、删除 | **大部分时间在这** |
| **插入模式** Insert | 按 `i` 进入。打字写代码 | 只在需要输入文字时 |
| **可视模式** Visual | 按 `v` 进入。选择文本 | 选中后复制/删除 |
| **命令行模式** | 按 `:` 进入。输入命令 | `:w` 保存 `:q` 退出 |

**新手最容易忽略的**：普通模式下按 `jj` 可以从插入模式退出——比伸手够 `Esc` 快多了。

## 3. 打开文件与项目

### 启动

```bash
nvim                    # 打开空白界面
nvim .                  # 打开当前目录（进入 dashboard）
nvim main.cpp           # 打开特定文件
nvim src/               # 打开目录（可以浏览）
```

### 启动后看到的界面（Dashboard）

打开 nvim 后你会看到一个**科幻 HUD 风格**欢迎页（NVIM-PIONEER vNext SYSTEM ONLINE），上面显示：
- Neovim 版本、插件数量、启动时间
- 最近 3 次 git 提交
- 快捷入口：`f` 找文件, `g` 全文搜索, `t` 切主题, `s` 会话列表, `u` 更新, `q` 退出

## 4. 搜索一切（Telescope）

这是最常用的功能。按 `<leader>ff`（Space + f + f）开始。

### 搜索快捷键

| 快捷键 | 功能 | 场景 |
|--------|------|------|
| `<leader>ff` | **查找文件** | 最常用，打开项目里的文件 |
| `<leader>fg` | **全文搜索** | 搜代码里的关键字 |
| `<leader>fb` | **切换已打开的文件** | 在多个文件间跳转 |
| `<leader>fr` | 最近文件 | 找回刚才关掉的 |
| `<leader>f;` | Git 文件 | 只看 Git 里的文件 |
| `<leader>fh` | 搜索 Vim 帮助 | 学 vim 命令 |
| `<leader>fw` | 搜索光标下单词 | 不用打字 |
| `<leader>fd` | 所有错误列表 | 看全项目的编译错误 |
| `<leader>fs` | 文件符号 | 当前文件的函数/变量列表 |
| `<leader>f.` | 恢复上次搜索 | 切回来继续搜 |

### 搜索结果里怎么操作

| 按键 | 功能 |
|------|------|
| `<C-j>` / `<C-k>` | 上下选择 |
| `Enter` 或 `<C-l>` | 打开文件 |
| `<C-v>` | **垂直分屏**打开 |
| `<C-s>` | **水平分屏**打开 |
| `<C-t>` | 新标签页打开 |
| `q` | 关闭搜索 |

## 5. 闪电跳转（Flash）

这是 vNext 最重要的新能力——**不用鼠标，不用逐行滚动，直接跳到屏幕上的任何位置**。

| 快捷键 | 模式 | 功能 |
|--------|:----:|------|
| `s` | n/x/o | **闪电跳转**——按 `s` 后所有匹配位置出现彩色字母标签，直接按标签字母跳过去 |
| `S` | n/x/o | 语法树跳转——在所有函数/类/变量节点之间跳 |
| `r` | o | 远程操作——先选范围再操作 |
| `R` | o/x | 语法树搜索——在语法树范围内搜索并操作 |

**举例**：光标在文件开头，想跳到下面某个 `return`。按 `s`，然后按 `re`（因为 "return" 的前两个字母弹出蓝色标签），瞬间到达。

## 7. 目录编辑（oil）

Oil 是另一种操作文件的方式——**把文件夹当文本编辑**。适合重命名、批量删除。

| 快捷键 | 功能 |
|--------|------|
| `<leader>o` | 打开 oil（当前目录） |
| `-` | 回到上级目录 |

**在 oil 里：**
- `Enter` → 打开文件
- `dd` → 删除文件/目录
- `yy` → 复制文件
- `p` → 粘贴
- `R` → 重命名
- `g.` → 切换隐藏文件显示
- `gx` → 用系统默认程序打开

| JavaScript/TypeScript | ts_ls |
| Lua | lua_ls |
| HTML/CSS/JSON | html/cssls/jsonls |
| Markdown | marksman |

| 快捷键 | 功能 |

| `gD` | 跳转声明 |

## 11. 语法树操作（Treesitter）

这是 Neovim 的核心优势之一——**对代码结构进行操作**，而不是对文本。

### 文本对象：选函数/类/参数

| 按键 | 选中什么 |
|------|---------|
| `yaf` | 复制**整个函数**（包括函数头和尾） |
| `daf` | 删除整个函数 |
| `cif` | 修改函数**内部**内容 |
| `yac` | 复制整个类 |
| `cia` | 修改函数参数 |
| `yab` | 复制整个代码块 |

> 格式：动词（`y`复制 `d`删除 `c`修改）+ `a`包含边界 / `i`内部 + `f`函数 / `c`类 / `a`参数 + `b`代码块

### 在函数/类之间跳转

| 按键 | 跳到 |
|------|------|
| `]m` | **下一个**函数开头 |
| `[m` | **上一个**函数开头 |
| `]]` | 下一个类开头 |
| `[[` | 上一个类开头 |
| `]M` / `[M` | 下/上一个函数结尾 |

### 交换参数位置

| 按键 | 功能 |
|------|------|
| `<leader>a` | 把当前参数和后一个**交换** |
| `<leader>A` | 把当前参数和前一个交换 |

### 增量选择

| 按键 | 功能 |
|------|------|
| `<c-space>` | 从光标处开始**逐级扩大选中范围** |

示例：光标放在函数名上 → 按 `<c-space>` → 选中函数名 → 再按 → 选中整个函数 → 再按 → 选中类

## 12. 保存自动格式化（Conform）

| Rust | rustfmt |
| JS/TS/JSX/TSX | prettier |
| HTML/CSS/JSON/YAML | prettier |
| Markdown | prettier |
| Shell | shfmt |

## 13. 顶部的文件标签栏（Bufferline）

打开多个文件后，屏幕**顶部会显示标签栏**，类似 Eclipse/VS Code。每个标签上还会显示 **diagnostics 标记**（错误数/警告数）。

```
▎ 1 init.lua  ● ▎ 2 lsp.lua  󰅚2 ▎ 3 main.cpp  ▎ 󰅖
────────────────────────────────────────────────────
```

- `▎` 粗竖线 = 当前激活文件
- `●` = 文件已修改未保存
- `󰅚2` = 2 个错误
- `1` `2` `3` = buffer 编号

| 快捷键 | 功能 |
|--------|------|
| `<Tab>` | 切换到**下一个**文件 |
| `<S-Tab>` | 切换到**上一个**文件 |
| `<leader>x` | 关闭当前文件 |
| `<leader>X` | 关闭全部文件 |
| `<leader>bC` | 关闭其他所有文件 |
| `<leader>bp` | 固定标签（不被关闭） |
| `Alt+,` / `Alt+.` | 左移/右移标签 |

## 14. 主题与界面

### 主题切换

| 快捷键 | 功能 |
|--------|------|
| `<leader>th` | **循环切换主题** |

**4 套主题：**
1. `gruvbox`（暖色复古）— 默认
2. `nightfox`（冷色现代）
3. `rose-pine`（柔和粉彩）
4. `dracula`（暗紫经典）

主题会记住上一次的选择，下次启动自动恢复。

### 界面增强（vNext 新增）

这套配置升级了整体 UI，统一了设计语言：

| 增强 | 效果 |
|------|------|
| **noice.nvim** | 命令行变成圆角浮动窗口，消息以 mini 模式在右下角显示 |
| **dressing.nvim** | 选择/输入对话框全部美化，统一圆角边框 |
| **incline.nvim** | 每个窗口右上角显示浮动文件名 + 诊断数量 |
| **mini.animate** | 光标移动有轨迹动画，滚动/分屏有平滑过渡 |
| **nvim-treesitter-context** | 滑动到函数内部时，顶部固定显示父函数名 |

### 状态栏监控

| 快捷键 | 功能 |
|--------|------|
| `<leader>tm` | 开关 CPU/RAM 监控 |
| `:ToggleEncoding` | 开关编码显示 |
| `:ToggleShortcuts` | 开关 Ctrl+C/V/A 标准快捷键 |

## 15. 全局搜索替换（Spectre）

vNext 新增——**不用终端 sed/rg，直接在 nvim 里可视化搜索替换**。

| 快捷键 | 功能 |
|--------|------|
| `<leader>sr` | **打开**全局搜索替换面板 |
| `<leader>sw` | 搜索光标下单词（globally） |
| `<leader>sp` | 当前文件内搜索 |

面板里可以选择哪些文件需要替换，实时预览结果，确认后批量执行。

## 16. 会话管理（persisted.nvim）

vNext 新增——**退出自动保存，下次启动自动恢复**。按 git 分支隔离，不同分支有独立的文件布局。

| 快捷键 | 功能 |
|--------|------|
| `<leader>qs` | 保存当前会话 |
| `<leader>ql` | 加载上次会话 |
| `<leader>qf` | 浏览所有保存的会话 |
| `<leader>qd` | 删除当前会话 |

不需要任何操作——退出 nvim 时自动保存所有打开的文件和窗口布局，下次 `nvim` 启动时自动恢复。

### 配置结构

```
~/.config/nvim/
├── init.lua                    ← 入口
├── CHEATSHEET.md               ← 快捷键速查
├── docs/
│   ├── README.md               ← 文档索引
│   └── guide/
│       ├── keybindings_CN.md   ← 快捷键大全
│       ├── keybindings.md      ← (EN)
│       ├── usage-guide_CN.md   ← 本文件
│       └── usage-guide.md      ← (EN)
└── lua/config/
    ├── options.lua             ← 基础设置 + 设计令牌
    ├── keymaps.lua             ← 全局快捷键
    ├── lazy.lua                ← 插件管理器
    ├── theme.lua               ← 主题逻辑
    ├── shortcuts.lua           ← Ctrl+C/V/A 模式
    └── plugins/
        ├── coding/             ← 编码（blink/lsp/rust）
        ├── editor/             ← 编辑增强（flash/treesitter/conform/...）
        ├── embedded/           ← 嵌入式（clangd/cmake/dap）
        ├── tools/              ← 工具（telescope/trouble/spectre/persistence/...）
        └── ui/                 ← 界面（lualine/noice/dressing/incline/animate/...）
```
