## 1. 这是什么

这是一套个人定制的 Neovim 配置，兼顾**功能完整**和**手感流畅**。

**一句话总结**：打开后按 `<Space>`（空格）是大部分操作的起点。

**核心功能一览：**
| 功能 | 用什么实现的 |
|------|------------|
| 文件搜索 | telescope.nvim（模糊搜索，带预览） |
| 代码补全 | blink.cmp（Rust 编写，极速） |
| 代码智能 | nvim-lspconfig + Mason（自动装 LSP） |
| 语法高亮 | nvim-treesitter（基于语法树，更准） |
| 格式化 | conform.nvim（保存自动格式化） |
| 文件浏览 | neo-tree + oil（双模式文件管理） |
| 顶部标签栏 | bufferline.nvim（类似 Eclipse/VS Code） |
| Git 集成 | gitsigns（行内改动标记） |
| 主题切换 | 4 套主题循环（gruvbox / nightfox / rose-pine / dracula） |
| 调试 | nvim-dap（F5/F10/F11/F12 调试） |
| 状态栏 | lualine（带 CPU/RAM 实时监控） |
| Markdown | 浏览器预览 + 编辑器内渲染 |

## 2. 基本概念：Mode 模式

Neovim 跟 VS Code/Eclipse 最大的不同是它有**模式**：

| 模式 | 说明 | 什么时候用 |
|------|------|-----------|
| **普通模式** Normal | 按 `Esc` 进入。移动光标、复制粘贴、删除 | **大部分时间在这** |
| **插入模式** Insert | 按 `i` 进入。打字写代码 | 只在需要输入文字时 |
| **可视模式** Visual | 按 `v` 进入。选择文本 | 选中后复制/删除 |
| **命令行模式** | 按 `:` 进入。输入命令 | `:w` 保存 `:q` 退出 |

**新手最容易忽略的**：普通模式下按 `jk` 或 `kj` 可以从插入模式退出——比伸手够 `Esc` 快多了。

## 3. 打开文件与项目

### 启动
```bash
nvim                    # 打开空白界面
nvim .                  # 打开当前目录（进入 dashboard）
nvim main.cpp           # 打开特定文件
nvim src/               # 打开目录（可以浏览）
```

### 启动后看到的界面（Dashboard）
打开 nvim 后你会看到一个欢迎页，上面有快捷键：
- 按 `u` → 更新插件
- 按 `f` → 搜索文件
- 按 `d` → 打开 dotfiles

### 基础操作
| 动作 | 按键 |
|------|------|
| 进入插入模式（开始打字） | `i` |
| 退出到普通模式 | `jk` 或 `kj` 或 `Esc` |
| 保存文件 | `<C-s>`（或 `:w Enter`） |
| 退出 | `:q Enter` |
| 不保存退出 | `:q! Enter` |
| 撤销 | `<C-z>` 或 `u` |
| 重做 | `<C-r>` |
| 复制选中文本 | 选中后 `y`（yank） |
| 粘贴 | `p` |
| 删除一行 | `dd` |
| 删除当前字符 | `x` |

## 4. 搜索一切（Telescope）

这是最常用的功能。按 `<leader>ff`（Space + f + f）开始。

### 搜索快捷键
| 快捷键 | 功能 | 场景 |
|--------|------|------|
| `<leader>ff` | **查找文件** | 最常用，打开项目里的文件 |
| `<leader>fg` | **全文搜索** | 搜代码里的关键字 |
| `<leader>fb` | **切换已打开的文件** | 在多个文件间跳转 |
| `<leader>fr` | 最近文件 | 找回刚才关掉的 |
| `<leader>f;` | Git 文件 | 只看 Git 里的文件 |
| `<leader>fh` | 搜索 Vim 帮助 | 学 vim 命令 |
| `<leader>fw` | 搜索光标下单词 | 不用打字 |
| `<leader>fd` | 所有错误列表 | 看全项目的编译错误 |
| `<leader>f.` | 恢复上次搜索 | 切回来继续搜 |

### 搜索结果里怎么操作
| 按键 | 功能 |
|------|------|
| `<C-j>` / `<C-k>` | 上下选择 |
| `Enter` 或 `<C-l>` | 打开文件 |
| `<C-v>` | **垂直分屏**打开 |
| `<C-s>` | **水平分屏**打开 |
| `<C-t>` | 新标签页打开 |
| `q` | 关闭搜索 |

## 6. 目录编辑（oil）

Oil 是另一种操作文件的方式——**把文件夹当文本编辑**。适合重命名、批量删除。

| 快捷键 | 功能 |
|--------|------|
| `<leader>o` | 打开 oil（当前目录） |
| `-` | 回到上级目录 |

**在 oil 里：**
- `Enter` → 打开文件
- `dd` → 删除文件/目录
- `yy` → 复制文件
- `p` → 粘贴
- `R` → 重命名
- `g.` → 切换隐藏文件显示

## 7. 代码补全（blink.cmp）

当你打字时，补全菜单会自动弹出。

| 快捷键 | 功能 |
|--------|------|
| `<C-Space>` | 手动触发补全 |
| `<Tab>` | 选择下一项 |
| `<S-Tab>` | 选择上一项 |
| `Enter` | 确认选择 |
| `<C-b>` / `<C-f>` | 上下滚动文档 |
| `<C-e>` | 取消补全 |

**补全来源：**
- LSP（语法感知，最准）
- 当前文件的文字（buffer）
- 文件路径
- 代码片段（snippets）

## 8. LSP：代码智能

打开代码文件后，LSP 会自动启动。支持的语言：

| 语言 | LSP 服务器 |
|------|-----------|
| Python | pyright |
| C/C++ | clangd |
| Rust | rust-analyzer |
| Go | gopls |
| JavaScript/TypeScript | ts_ls |
| Lua | lua_ls |
| HTML/CSS/JSON | html/cssls/jsonls |
| Markdown | marksman |

### LSP 快捷键
| 快捷键 | 功能 |
|--------|------|
| `gd` | **跳转定义** |
| `gr` | **查找引用** |
| `gD` | 跳转声明 |
| `K` | **查看文档**（光标放变量上按 K） |
| `<leader>rn` | **重命名**变量/函数 |
| `<leader>ca` | **代码操作**（修复建议） |
| `<leader>d` | 查看当前行错误 |
| `]d` / `[d` | 跳到下/上一个错误 |
| `<leader>th` | 开关类型标注（inlay hints） |

## 9. 语法树操作（Treesitter）

这是 Neovim 的核心优势之一——**对代码结构进行操作**，而不是对文本。

### 文本对象：选函数/类/参数

| 按键 | 选中什么 |
|------|---------|
| `yaf` | 复制**整个函数**（包括函数头和尾） |
| `daf` | 删除整个函数 |
| `cif` | 修改函数**内部**内容 |
| `yac` | 复制整个类 |
| `cia` | 修改函数参数 |
| `yia` | 复制参数 |
| `yab` | 复制整个代码块 |

> 格式：动词（`y`复制 `d`删除 `c`修改）+ `a`包含边界 / `i`内部 + `f`函数 / `c`类 / `a`参数

### 在函数/类之间跳转

| 按键 | 跳到 |
|------|------|
| `]m` | **下一个**函数开头 |
| `[m` | **上一个**函数开头 |
| `]]` | 下一个类开头 |
| `[[` | 上一个类开头 |

### 交换参数位置

| 按键 | 功能 |
|------|------|
| `<leader>a` | 把当前参数和后一个**交换** |
| `<leader>A` | 把当前参数和前一个交换 |

### 增量选择

| 按键 | 功能 |
|------|------|
| `<c-space>` | 从光标处开始**逐级扩大选中范围** |

示例：光标放在函数名上 → 按 `<c-space>` → 选中函数名 → 再按 → 选中整个函数 → 再按 → 选中类

## 10. 保存自动格式化（Conform）

**保存文件时自动格式化代码**，无需手动触发。

**手动触发**：按 `<leader>fm`

支持的文件类型和格式化工具：

| 文件类型 | 工具 |
|---------|------|
| Lua | stylua |
| Python | ruff |
| C/C++ | clang-format |
| Rust | rustfmt |
| JS/TS/JSX/TSX | prettier |
| HTML/CSS/JSON/YAML | prettier |
| Markdown | prettier |
| Shell | shfmt |
| Terraform | terraform_fmt |

## 11. 顶部的文件标签栏（Bufferline）

打开多个文件后，屏幕**顶部会显示标签栏**，类似 Eclipse/VS Code。

```
▎ init.lua  ▎ telescope.lua  ▎ neo-tree.lua  ▎ 󰅖
────────────────────────────────────────────────────
```

| 快捷键 | 功能 |
|--------|------|
| `<Tab>` | 切换到**下一个**文件 |
| `<S-Tab>` | 切换到**上一个**文件 |
| `<leader>x` | 关闭当前文件 |
| `<leader>b` | 新建空白文件 |
| `<leader>bC` | 关闭其他所有文件 |
| `<leader>bp` | 固定标签（不被关闭） |
| `Alt+,` / `Alt+.` | 左移/右移标签 |

## 12. 主题切换

| 快捷键 | 功能 |
|--------|------|
| `<leader>th` | **切换主题**（循环） |

**4 套主题：**
1. `gruvbox`（暖色复古）— 默认
2. `nightfox`（冷色现代）
3. `rose-pine`（柔和粉彩）
4. `dracula`（暗紫经典）

主题会记住上一次的选择，下次启动自动恢复。

## 13. 状态栏的 CPU/RAM 监控

底部状态栏右侧显示 CPU 使用率和 RAM 占用率，带颜色动画：
- 🟢 < 50% — 绿色
- 🟡 50-80% — 黄色
- 🔴 > 80% — 红色

| 快捷键 | 功能 |
|--------|------|
| `<leader>tm` | 开关 CPU/RAM 监控 |
| `:ToggleEncoding` | 开关编码显示 |

## 14. 调试代码（DAP）

支持 C/C++/Rust/Python 等语言的断点调试。

| 快捷键 | 功能 |
|--------|------|
| `<F5>` | 开始/继续调试 |
| `<F10>` | 单步跳过 |
| `<F11>` | 单步进入 |
| `<F12>` | 单步跳出 |
| `<leader>b` | 切换断点 |

调试界面会**自动打开**，结束时自动关闭。

## 15. Rust 专用功能

打开 Rust 文件（`.rs`）时，有以下额外功能：

| 快捷键 | 功能 |
|--------|------|
| `<leader>dr` | 查看可调试目标 |
| `K` | 带操作的 hover 文档 |

同时 `crates.nvim` 会在编辑 `Cargo.toml` 时提示版本更新。

## 16. 终端内嵌

可以在 nvim 里打开一个**浮动终端**，不用离开编辑器就能跑命令。

| 快捷键 | 功能 |
|--------|------|
| `<C-\>` | 开关浮动终端 |

在里面 `exit` 回车关闭。

## 17. CMake 集成

编辑 C/C++ 项目时有 CMake 支持：

| 快捷键 | 功能 |
|--------|------|
| `<leader>cg` | 生成 CMake（cmake generate） |
| `<leader>cb` | 编译 CMake（cmake build） |

也会自动生成 `compile_commands.json`，clangd LSP 用它做精准代码分析。

## 18. 贴士与常见操作

### 窗口分屏
| 快捷键 | 功能 |
|--------|------|
| `<leader>sv` | 垂直分屏 |
| `<leader>sh` | 水平分屏 |
| `<leader>h/j/k/l` | 在窗口间移动 |

### 文本操作
| 快捷键 | 功能 |
|--------|------|
| `<leader>nh` | 取消搜索高亮 |
| `<leader>lw` | 切换换行 |
| `n` / `N` | 搜索下一个/上一个匹配（自动居中） |
| `<C-d>` / `<C-u>` | 翻半页并居中 |
| `Alt+j` / `Alt+k` | 移动选中行（可视模式） |

### 标准快捷键切换
这套配置默认开启了 `Ctrl+C` / `Ctrl+V` / `Ctrl+A` / `Ctrl+Z` / `Ctrl+S` 快捷键，让你像 VS Code 一样操作：

| 快捷键 | 功能 |
|--------|------|
| `<C-c>` | 复制（选中文本后） |
| `<C-v>` | 粘贴 |
| `<C-a>` | 全选 |
| `<C-z>` | 撤销 |
| `<C-s>` | 保存 |

如果不想用，输入 `:ToggleShortcuts` 一键切回原生 Vim 风格。

### 保存/恢复工作区
| 快捷键 | 功能 |
|--------|------|
| `<leader>ss` | 保存当前所有打开的窗口和文件 |
| `<leader>sl` | 恢复上次的工作区 |

```
├── CHEATSHEET.md      ← 快捷键速查表
├── docs/              ← 完整文档
│   ├── README.md
│   └── guide/
│       ├── keybindings_CN.md
│       ├── keybindings.md
│       └── usage-guide_CN.md   ← 本文件
└── lua/config/
    ├── options.lua    ← 基础设置
    ├── keymaps.lua    ← 全局快捷键
    ├── lazy.lua       ← 插件管理器
    ├── theme.lua      ← 主题逻辑
    ├── shortcuts.lua  ← Ctrl+C/V/A 切换
    └── plugins/
        ├── coding/    ← 编码相关
        ├── editor/    ← 编辑器增强
        ├── embedded/  ← 嵌入式开发
        ├── tools/     ← 工具
        └── ui/        ← 界面
```
